# 事務手順標準化ツール

タニコー豊橋の「営業所事務の標準化・シェアードサービス化」に合わせた、事務手順整理の静的サンプルです。営業所ごとの依頼、受発注、納品・請求前確認、問い合わせ対応を、誰でも同じ品質で進めるための受付情報・確認項目・手順書・FAQへ落とし込む考え方を示します。

このサンプルは、特定の実データを再現したものではなく、事務作業で起こりやすい受付、確認、記録、手順化の流れを想定して作成しています。実際の業務内容や社内ルールに応じて、入力欄、確認項目、出力文、進捗ステータスをカスタムできる前提です。

## 入っているツール

- 依頼の整理
- 帳票チェックアシスタント
- 手順書ドラフトメーカー
- 社内資料参照・回答下書き
- 期限・進捗ボード

各ツールを選ぶと、画面上部に「想定される使用場面」と「この画面でできること」が表示されます。
下段の入力画面で実際に操作すると、完了内容が黒い通知ラベルで表示されます。

## 設計意図

課題: 営業所ごとに受発注、納品・請求前確認、問い合わせ対応の確認手順が違うと、確認漏れや引き継ぎ漏れが起きやすい。

仕組み: 入力内容を、受付情報、確認項目、判断基準、回答文、記録形式に分解して表示します。

期待効果: 豊橋で整えた共通手順を、他営業所でも同じ品質で使える状態へ近づけます。

自分の工夫: 価格表整理で得た「正本を決め、参照ルールを固定し、現場が使いやすい形に戻す」経験を、営業所事務の標準化に翻訳しています。

## 社内資料参照の説明

専門用語は前面に出さず、「社内資料、最新版、承認者、例外条件を整理し、根拠つきで回答や確認メモの下書きを作る仕組み」と説明する想定です。

サンプル内では、価格差異の問い合わせ回答、見積確認メモの作成補助、納品前確認、請求前確認の下書き補助、手順書・FAQ化の5パターンを用意しています。出力は確定処理ではなく下書きで、価格、納品、請求、社外提示に関わる判断は必ず人が確認する前提です。

## 開き方

`index.html` をブラウザで開くと動きます。外部通信はありません。入力内容や参照データはすべてサンプルです。

## 応募での見せ方

「事務処理をする人」ではなく、「各営業所のナレッジを集め、豊橋から共通手順として発信できる人」として見せるための成果物です。
